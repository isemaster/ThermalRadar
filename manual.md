# TERMO1 Radar — руководство пилота

**Версия:** v0.0.5

---

## 1. Что это

TERMO1 — Android-приложение для парапланеризма. Анализирует микрораскачку крыла через акселерометр телефона, определяет **направление на термик** и показывает его на радарном экране. В слепом режиме — голосовые подсказки.

**Телефон крепится к парапланеристу** (на грудь, в карман подвески, на руку).

---

## 2. Требования к телефону

| Датчик | Обязательно? | Для чего |
|--------|-------------|----------|
| Акселерометр | **Да** | Основа работы |
| Барометр | Нет | Вариометр + автостарт полёта |
| Магнитометр | Нет | Компас (ориентация по сторонам света) |

**Минимально:** любой Android 8.0+ (API 26) с акселерометром.

---

## 3. Быстрый старт

### Первый запуск

1. Установить APK
2. Закрепить телефон на груди (горизонтально, экраном вверх)
3. Запустить TERMO1
4. Положить телефон неподвижно на 2 секунды — автокалибровка
5. Готово — радар показывает ПОИСК

### Перед полётом (в подвеске на турнике)

1. Повисели неподвижно 3 секунды
2. **Долгий тап** на кнопку **СТАРТ** (внизу слева)
3. Телефон запомнит угол крепления — появится строка:
   `Крепление: 35° | Крен: 32°`
4. Настройки → галочка **Слепой полёт (карман)** — если телефон в кармане

### Слепой режим

Телефон в кармане, экран тёмный, голос подсказывает:
- «Термик спереди 40 метров»
- «Термик справа 30 метров»
- «Термик слева 80 метров»

**Требование:** телефон должен лежать в кармане так, чтобы его левая сторона была слева от пилота. Наклон может быть любым.

## 4. Экран (обычный режим)

```
  [⨯]           [⚙]
  ┌─────────────────────────┐
  │   ┌─ ПОИСК ─────────┐   │  ← статус
  │   └─────────────────┘   │      ПОИСК / ПОДОЗРЕНИЕ / ТЕРМИК РЯДОМ /
  │                         │      ВНУТРИ ТЕРМИКА / НАБОР / СНИЖЕНИЕ
  │          +1.2           │  ← вариометр (м/с)
  │          м/с            │      зелёный = набор, красный = снижение
  │                         │
  │       ┌── N ──┐         │
  │       │       │         │
  │  W ───│   ●   │─── E    │  ← радар (30 FPS)
  │       │   ⬤   │         │      ● — пилот (центр)
  │       │       │         │      ⬤ — термик (жёлтый)
  │       └── S ──┘         │      N — север (компас)
  │  50м 100м 125м 150м     │      кольца: 50м, 100м, 125м, 150м
  │                         │
  │ ±0.12 м/с² | 1.35 Гц    │  ← данные с датчиков
  │ 045°                    │
  │ Крепл: 35° | Крен: 32°  │  ← наклон телефона (после калибровки)
  ├─────────────────────────┤
  │[СТАРТ]   пишем лог [СТОП]│  ← кнопки
  │ 45м            45м       │  ← AGL слева, MSL справа
  │     +00:12:34            │  ← время полёта
  │      12:34:56            │  ← системное время
  │        [ТЕСТ]            │  ← тест датчиков
  └─────────────────────────┘
```

### Элементы

**Статус-пилл** (вверху по центру):
- ПОИСК — термиков нет, радар чист
- ПОДОЗРЕНИЕ — слабый сигнал, направление нестабильно
- ТЕРМИК РЯДОМ — уверенный сигнал, есть направление
- ВНУТРИ ТЕРМИКА — пилот в зоне подъёма
- НАБОР — варио > 1 м/с + турбулентность
- СНИЖЕНИЕ — варио < -1 м/с

**Радар** (центр):
- Вращается с компасом (N всегда сверху)
- Жёлтый круг = термик. Чем ближе и ярче — тем увереннее сигнал
- Пунктирная линия от пилота к термику
- Кольца: 50м, 100м, 125м (пунктир), 150м

**Данные датчиков** (слева под радаром):
- `±0.12 м/с²` — уровень турбулентности
- `1.35 Гц` — частота микрораскачки
- `045°` — направление на термик от N
- `Крепление: 35°` — запомненный угол крепления (после калибровки)
- `Крен: 32°` — текущий наклон телефона (меняется в спирали)

**Высоты:**
- Слева — AGL (над землёй)
- Справа — MSL (над уровнем моря)

### Кнопки

| Кнопка | Где | Тап | Долгий тап |
|--------|-----|-----|------------|
| ⨯ | Верх-слева | Выход | — |
| ⚙ | Верх-справа | Настройки | — |
| СТАРТ | Внизу слева | Старт записи лога | Сброс калибровки |
| СТОП | Внизу справа | Стоп записи лога | — |
| ТЕСТ | Под высотами | Тест датчиков (6 шагов) | — |

## 5. Режимы детекции

| Статус | Уровень (g) | Дистанция | Что происходит |
|--------|------------|-----------|----------------|
| ПОИСК | < 0.005 | > 150 м | Нет сигнала |
| ПОДОЗРЕНИЕ | 0.005–0.020 | 75–150 м | Слабый сигнал |
| ТЕРМИК РЯДОМ | 0.020–0.080 | 37–75 м | Стабильное направление |
| ВНУТРИ ТЕРМИКА | > 0.080 | < 37.5 м | Пилот в зоне подъёма |

После подтверждения (сигнал стабилен 2+ периода) **направление блокируется** — шарик на радаре не прыгает.

## 6. Звук и вибрация

### Вариометр (Brauniger IQ)

| Набор | Звук |
|-------|------|
| 0.3 м/с | Длинный низкий бип + пауза |
| 3.0 м/с | Средний бип + пауза |
| 9.0 м/с | Короткий высокий бип + пауза |
| > 9 м/с | Непрерывный тон 1000 Гц |
| Снижение | Тишина |

### Голосовые подсказки (TTS)

При обнаружении нового термика в полёте:
- «Термик спереди 30 метров»
- «Термик спереди справа 60 метров»
- «Термик слева 80 метров»
- «Термик спереди слева 40 метров»
- «Термик справа 50 метров»

Задние сектора (сзади) — молчок. Дебаунс 8 секунд на повтор.

### Вибрация

При смене статуса на **ТЕРМИК РЯДОМ** или **НАБОР** — виброимпульс 300 мс.

## 8. Логирование

### Ручной режим
- **СТАРТ** — начать запись
- **СТОП** — завершить

### Автоматический режим
- **Автостарт:** высота изменилась на 10 м за 10 секунд
- **Автостоп:** высота не меняется ±2 м в течение 5 минут

### Формат файлов

Лог режется по 10 минут. Имена файлов:

```
Start20260531150000-20260531151000   (первый чанк)
Flight20260531151000-20260531152000  (середина)
Finish20260531153000-20260531153034  (последний, < 10 мин)
```

Каждый файл — ZIP-архив, внутри CSV с теми же данными (без заголовков).

**Колонки (27):**
```
dtMs, gpsSpeed, gpsHeading, gpsLat, gpsLon, gpsAlt,
ax, ay, az, gx, gy, gz, mx, my, mz,
pressure, pitch, roll, heading,
thermalAngle, thermalStrength, thermalDist, thermalSource, snr, noiseFloor
```

### Где лежат логи
`Android/data/com.termo1.radar/files/logs/`

Отправка: Настройки → «Отправить логи» → выбрать файлы → отправить.

## 9. Симуляция

Настройки → **Симуляция 75 с**.

Сценарий: полёт → появление термика справа → поворот → спираль 35 секунд → выход.

## 10. Тест датчиков

Кнопка **ТЕСТ** — 6 шагов с обратной связью:

| Шаг | Действие | Что проверяется |
|-----|---------|-----------------|
| 1 | Положить неподвижно на 5 с | Калибровка шума |
| 2 | Качать влево-вправо | Акселерометр, ось X |
| 3 | Качать вперёд-назад | Акселерометр, ось Y |
| 4 | Повернуть на 90° влево | Компас |
| 5 | Крутить 360° с наклоном | Компас + акселерометр |
| 6 | Трясти 3–5 Гц | Полоса фильтра |

---

## 11. Калибровка наклона

При первом использовании в подвеске:

1. Повисели на турнике неподвижно 3 секунды
2. **Долгий тап** на **СТАРТ** (внизу слева)
3. Через 2 секунды на экране появится:
   `Крепление: 35° | Крен: 32°`
4. Угол сохраняется — повторно калибровать не нужно

**Перекалибровка:** долгий тап на СТАРТ в любой момент.

---

## 12. Советы пилоту

1. **Перед полётом:** закрепить телефон жёстко — болтающийся телефон даёт ложный сигнал
2. **Калибровка:** на турнике, повисеть неподвижно 3 сек, долгий тап СТАРТ
3. **В кармане:** включить «Слепой полёт» в настройках, левый край телефона — слева пилота
4. **Шарик на радаре** показывает направление, КУДА лететь
5. **Яркий жёлтый** — уверенный сигнал, бледный — слабый
6. **Большой шарик** — широкая вибрация (0.5 Гц), маленький — узкая (2.5 Гц)
7. **Голос** подсказывает направление и дистанцию в полёте
8. **Логи** отправляйте из настроек для анализа
