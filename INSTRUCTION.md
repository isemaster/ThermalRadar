## 1. ЧТО ЭТО

TERMO1 — Android-приложение для парапланеризма. Анализирует микрораскачку крыла через **акселерометр телефона**, определяет **направление на термик** и показывает его на радарном экране.

**Телефон крепится к парапланеристу** (на грудь, в карман harness'а).

---

## 2. ТРЕБОВАНИЯ К ТЕЛЕФОНУ

| Датчик | Обязательно? | Для чего |
|--------|-------------|----------|
| Акселерометр (TYPE_ACCELEROMETER) | **Да** | Компас + детектор (если нет LINEAR) |
| Акселерометр линейный (TYPE_LINEAR_ACCELERATION) | Нет | Детектор микрораскачки (точнее) |
| Барометр (TYPE_PRESSURE) | Нет | Вариометр (без него — только GPS высота) |
| Магнитометр (TYPE_MAGNETIC_FIELD) | Нет | Компас (без него — курс по GPS) |
| Гироскоп (TYPE_GYROSCOPE) | Нет | Журналирование |
| GPS | Нет | Скорость, курс, высота |

**Минимальный:** акселерометр (встроен во все Android).
**Чувствительность:** LSM6DSL (Samsung A8+) — 90 µg/√Hz, достаточно. Любой современный телефон работает.

## 3. ЗАПУСК

1. Установить APK
2. Запросить разрешение на GPS (только для лога полёта)
3. Телефон автоматически переходит в полноэкранный режим
4. **Положить телефон неподвижно на 2 секунды** — калибровка шума

## 4. ЧТО НА ЭКРАНЕ

```
  [⨯]           [⚙]
  ┌─────────────────────────┐
  │    ┌─ ПОДОЗРЕНИЕ ─┐     │  ← статус
  │    └──────────────┘     │
  │                         │
  │          +1.2           │  ← вариометр (м/с)
  │          м/с            │
  │                         │
  │       ┌── N ──┐         │
  │       │       │         │
  │  W ───│   ●   │─── E    │  ← радар (вращается по компасу)
  │       │   ⬤   │         │     ● — пилот (центр)
  │       │       │         │     ⬤ — термик (жёлтый)
  │       └── S ──┘         │
  │  50м 100м 125м 150м     │  ← кольца дальности
  │                         │
  │ ±0.12 м/с²              │  ← данные с датчиков
  │ 1.35 Гц                 │
  │ 045° от N               │
  ├─────────────────────────┤
  │[КАЛИБР]                  │
  │ 45м            45м       │  ← AGL слева, MSL справа
  │     +00:12:34            │  ← время полёта
  │      12:34:56            │  ← системное время
  └─────────────────────────┘
```

### Поля:

1. **Вариометр** (крупная цифра вверху) — вертикальная скорость в м/с. Зелёный = набор, красный = снижение.
2. **Статус-пилл** (над варио): ПОИСК / ПОДОЗРЕНИЕ / ТЕРМИК РЯДОМ / ВНУТРИ ТЕРМИКА
3. **Радар** (центр):
   - Вращается так, что N (север) всегда сверху
   - Кольца: 50м, 100м, 125м (пунктир), 150м (граница)
   - Жёлтый круг = термик. Чем ближе и ярче — тем увереннее сигнал
   - Пунктирная линия от пилота к термику
4. **Данные датчиков** (слева под радаром): турбулентность (м/с²), частота (Гц), направление (°)
5. **Высоты**: AGL (над землёй) слева, MSL (над уровнем моря) справа
6. **Время полёта** и **системное время**

### Кнопки:

| Кнопка | Где | Действие |
|--------|-----|----------|
| ⨯ | Верх-слева | Выход (с подтверждением) |
| ⚙ | Верх-справа | Настройки |
| КАЛИБР | Под варио слева | Сброс калибровки (если шум изменился) |
| СТАРТ/СТОП | Под варио справа | Вкл/выкл запись лога |
| ТЕСТ | Под высотами | Пошаговый тест датчиков (6 шагов) |
| SIM | В левом верхнем углу (только в симуляции) | |

## 5. РЕЖИМЫ ДЕТЕКЦИИ

| Статус | Уровень (g) | Дистанция | Что происходит |
|--------|------------|-----------|----------------|
| ПОИСК | < 0.005 | > 150 м | Термиков нет, радар чист |
| ПОДОЗРЕНИЕ | 0.005–0.020 | 75–150 м | Есть сигнал, направление нестабильно |
| ТЕРМИК РЯДОМ | 0.020–0.080 | 37–75 м | Сигнал стабилен, показано направление |
| ВНУТРИ ТЕРМИКА | > 0.080 | < 37.5 м | Пилот в зоне подъёма |

После подтверждения термика (сигнал стабилен 2+ периода) **направление блокируется** — шарик на радаре не прыгает.

## 6. ЗВУК

Звук имитирует Brauniger IQ vario:

| Набор | Звук |
|-------|------|
| 0.3 м/с | Длинный низкий бип + длинная пауза |
| 3.0 м/с | Средний бип + средняя пауза |
| 9.0 м/с | Короткий высокий бип + короткая пауза |
| > 9 м/с | Непрерывный тон 1000 Гц |
| Снижение | Тишина |
| Обнаружен термик | Одиночный бип 800 Гц × 150 мс |

Звук отключается в настройках.

## 8. ЛОГИРОВАНИЕ

### Ручной режим
Нажать **СТАРТ** — начнётся запись всех датчиков 50 Гц. Повторное нажатие **СТОП** — сохранение.

### Автоматический режим
В настройках **Лог полёта СТАРТ**. Запись начинается автоматически когда:
- Варио > 1 м/с ИЛИ
- GPS скорость > 3 м/с ИЛИ
- Турбулентность > 0.05g

Запись останавливается через 5 минут без сигнала полёта.

### CSV формат (22 колонки)

```
dtMs,ax,ay,az,gx,gy,gz,mx,my,mz,pressure,pitch,roll,
heading,gpsSpeed,gpsHeading,
thermalAngle,thermalStrength,thermalDist,thermalSource,
snr,noiseFloor
```

| Колонка | Ед.изм | Описание |
|---------|--------|----------|
| dtMs | мс | Время от старта записи |
| ax,ay,az | mG | Линейное ускорение |
| gx,gy,gz | mdps | Гироскоп |
| mx,my,mz | μT | Магнитометр |
| pressure | hPa | Давление |
| pitch,roll | рад | Наклон/крен |
| heading | ° | Курс (компас или GPS) |
| gpsSpeed | м/с | GPS скорость |
| gpsHeading | ° | GPS курс |
| thermalAngle | ° | Направление на термик |
| thermalStrength | — | Сила сигнала (1–8) |
| thermalDist | м | Дистанция до термика |
| thermalSource | строка | accel / simulated |
| snr | — | SNR турбулентности |
| noiseFloor | g | Текущий шумовой порог |

### Размер лога
- 1 час полёта: ~9 МБ CSV → ~1.5 МБ ZIP (сжимается автоматически)

## 9. СИМУЛЯЦИЯ

В настройках → **Симуляция 75 с**.

Сценарий:
- 0–5 с: полёт на север, снижение -1 м/с
- 5 с: термик появляется справа (100 м)
- 5–15 с: поворот к термику
- 15–50 с: спираль внутри термика, набор +2.5–3.5 м/с
- 50–75 с: выход, набор затухает до 0

По окончании — возврат в настройки.

## 10. ТЕСТИРОВАНИЕ ДАТЧИКОВ

Кнопка **ТЕСТ** (под высотами) — 6 шагов с обратной связью:

| Шаг | Действие | Что проверяется |
|-----|---------|----------------|
| 1 | Положить неподвижно на 5 с | Калибровка |
| 2 | Качать влево-вправо (ось X) | Акселерометр |
| 3 | Качать вперёд-назад (ось Y) | Акселерометр |
| 4 | Повернуть на 90° влево | Компас |
| 5 | Крутить 360° с наклоном | Компас + акселерометр |
| 6 | Трясти 3–5 Гц | Полоса пропускания фильтра |

На каждом шаге — real-time подсказки зелёным/жёлтым/красным текстом. После прохождения всех шагов — диалог с предложением отправить лог.

## 11. АЛГОРИТМ ДЕТЕКЦИИ

```
Сигнал с LINEAR_ACCEL (50 Гц)
  │
  ▼
Полосовой фильтр Баттерворта 2-го порядка
  ├── ФВЧ 0.25 Гц (убирает дрейф, повороты)

  │
  ▼
Zero-crossing детектор → частота (0.5–2.5 Гц)
  │
  ▼
Сравнение с порогами (SUSPECT / THERMAL / INSIDE)
  │
  ▼
Верификация: 2 периода сигнала → ratio level
  ├── ratio ≥ 1.2 → CONFIRMED (термик)
  ├── ratio ≤ 0.8 → REJECTED
  └── 3× REJECTED → "ЛЕТАЙ ЗМЕЙКОЙ"
  │
  ▼
Дистанция: 150 × √(0.05 / RMS_мс²)
Размер шарика: 0.5 Гц → большой, 2.5 Гц → маленький
  │
  ▼
Радар (Canvas, 30 FPS)
```

## 12. СОВЕТЫ ПИЛОТУ

1. **Перед полётом:** положить телефон на 2 с неподвижно для калибровки
2. **Телефон крепить жёстко** к harness'у — болтающийся телефон даёт ложный сигнал
3. **Шарик на радаре** показывает направление, КУДА лететь
4. **Яркий жёлтый** — уверенный сигнал, бледный — слабый/старый
5. **Большой шарик** — широкая вибрация (0.5 Гц), маленький — узкая (2.5 Гц)
6. Если долго нет термиков и радар показывает хаотичные blip — нажмите **КАЛИБР** (сброс шума)
7. **"ЛЕТАЙ ЗМЕЙКОЙ"** — 3 раза подряд сигнал затухал при приближении. Летите змейкой, меняйте курс
8. Логи отправляйте из настроек для анализа
