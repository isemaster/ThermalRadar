# Калибровка наклона — запоминание полётного положения

## Сценарий калибровки

1. Пилот вешает подвеску на турник
2. Запускает TERMO1, закрепляет телефон на груди
3. Нажимает **долгим тапом** СТАРТ → сброс калибровки наклона
4. Виснет неподвижно 3 секунды
5. Прибор запоминает: `mountTiltDeg = 35°` (угол крепления от горизонта)
6. Угол сохраняется в SharedPreferences — не сбрасывается при перезапуске
7. На экране: `Крепление: 35° | Тек: 32°` (mount vs current)

## Что меняется в полёте

- **Крен в спирали** — телефон наклоняется вбок на 20–40°
  currentTilt растёт, mountTilt остаётся прежним
  Коррекция порогов идёт по mountTilt — стабильна
- **Смена позы** — пилот наклонился в harness'е
  currentTilt дрейфует, mountTilt не меняется
- **Перемонтирование телефона** — новый угол, нужна перекалибровка

## Формула угла (3D)

```
tiltDeg = acos(gz / |g|)  — угол оси Z телефона от вертикали мира

0°  = экран вверх (горизонтально, как на столе)
45° = экран под 45° (типичное крепление на груди)
90° = экран вертикально (карман harness'а)
```

## Коррекция порогов (static, по mountTilt)

Горизонтальное колебание амплитудой A проецируется на плоскости
экрана как A × cos(tilt). При tilt=45° теряем 30% амплитуды.

```
tiltCorrection = 1.0 / cos(mountTiltDeg_rad)
  0°  → 1.00× (без коррекции)
  45° → 1.41× 
  60° → 2.00×
```

Пороги делятся на tiltCorrection — детектор становится чувствительнее.
