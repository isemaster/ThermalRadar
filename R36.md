## 0. Резюме одним абзацем

Автор проделал **исключительную работу** за 27 релизов. Все 9 Priority-0 блокеров и ~75% Priority-1 пунктов закрыты. Главный приоритет v0.2.13 — **переход на `TYPE_ROTATION_VECTOR`** (встроенный Kalman filter ядра Android) — оказался лучше, чем рекомендованный в ревью `TYPE_GRAVITY`: это промышленный fusion gyro+accel+mag, который даёт на порядок более стабильный компас в турбулентности. Добавлены три серьёзных модуля (TrackReplayer 573 строки, IgcLogger FAI 472 строки, CirclingManager 776 строк), IGC-логи приведены к FAI-совместимому формату с CRC-16-CCITT, реализована streaming-запись LogManager, takeoff/landing FSM с гистерезисом 3.0/1.0 м/с. Остались 5 мелких Priority-1 пунктов (не блокеры) + 1 архитектурный долг, который УСУГУБИЛСЯ: MainActivity вырос с 2 709 до **3 080 строк**, и при 12 461 строках product code тесты так и не выросли (149 строк — те же, что в v0.2.5). Для дальнейшего развития критически важно (а) закрыть оставшиеся мелочи, (б) начать разбиение God Object, (в) добавить тесты, (г) определить приоритеты product-функций: safety features (LD/reachability/MacCready), offline maps, интеграции (XCTrack/XContest), и (опционально) предиктивная аналитика термов на ML.

### 2.2. Новый архитектурный долг — MainActivity достиг 3 080 строк

```
v0.2.5:  2 709 строк
v0.2.9:  2 713 строк
v0.2.36: 3 080 строк  (+367 строк за 27 релизов)
```

В MainActivity теперь живут: lifecycle, render loop, sensor listener, GPS, TTS, IGC, CSV, FSM, circling, blind mode, simulation, **track replay UI**, **player panel UI**, **pilot config**, **seekbar handling**, **REC indicator**, calibration, settings, permissions, compass, map. Это уже не God Object, а God Universe.

Проблема: каждое новое изменение в одной из 15+ зон рискует сломать соседнюю. Любой bug-fix — это охота на тонкие side-effects. Ревью PR занимает в 3-5 раз дольше.

### 2.3. Тесты не выросли — критическая проблема

```
v0.2.5:  Termo1UnitTests.java — 150 строк (6 тестов на SimulationManager)
v0.2.36: Termo1UnitTests.java — 149 строк (те же 6 тестов)
```

При 12 461 строке product code — coverage 0% на всех критических путях:
- `HeadingFilter` (195 строк, edge cases: unwrap, deadband adaptive, turn rate clamp, reset)
- `VarioManager` (220 строк, calibration, baroOffset, GPS sync)
- `IgcLogger` (472 строки, CRC-16-CCITT — идеально testable!)
- `FlightStateMachine` (291 строка, takeoff/landing hysteresis)
- `TrackReplayer` (573 строки, midnight crossing, cos(lat), ZIP)
- `CirclingManager` (776 строк — самый крупный flight-модуль!)
- `LogManager` (383 строки, streaming, flush, ZIP)

**Особенно критично для IGC CRC-16-CCITT**: одна ошибка в полиноме или порядке байтов — и файл не пройдёт валидацию на XContest/XCTrack. Это должно быть покрыто golden file test'ом: загрузить эталонный IGC, проверить CRC совпадает.

### 2.4. VERSION файл рассинхронизирован

Файл `VERSION` описывает v0.2.13, git tag — v0.2.36. Это не баг, но сбивает с толку при bug report'ах. Рекомендация: либо генерировать VERSION из git tag (`git describe --tags` в build.gradle), либо обновлять вручную в каждом релизе.

## 5. Top-5 рекомендаций (если не читать всё остальное)

1. **Тесты на IgcLogger CRC-16-CCITT — обязательно.** Это golden file test: взять эталонный IGC, который проходит валидацию на XContest, прогнать через ваш CRC, сравнить. Если разойдётся хоть на бит — вы не сможете публиковать полёты на контестах. Это 4 часа работы, которые спасут вам десятки часов отладки в будущем.

2. **Glide Ratio + Reachability circle — это killer features.** Ни одно парапланерное Android-приложение (кроме XCSoar и XCTrack) не показывает reachability circle в реальном времени. Если вы это сделаете — у вас будет уникальное преимущество. ~16 часов работы.

3. **Разбить MainActivity на координаторы.** 3 080 строк — это путь в ад. Каждый следующий фикс рискует сломать соседний функционал. Начните с выделения `ReplayCoordinator` (track replay UI уже обособлен) и `LogCoordinator` (IGC + CSV + LogManager). ~40 часов.

4. **Offline regional tile packs.** Это то, что отличает авиационное приложение от туристического. Парапланеристы летают в горах, где нет мобильного интернета. Если тайлы не предзагружены — карты не работают. ~24 часа.

5. **LeakCanary + StrictMode + CI.** Это базовая инженерная гигиена, которая стоит 7 часов, но ловит 80% проблем с памятью, main thread блокировками и регрессиями. После настройки CI каждый PR автоматически проверяется.
