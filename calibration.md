# Калибровка TERMO1 с учётом наклона телефона в подвеске

**Версия:** v0.0.5

## Проблема

Телефон крепится на груди пилота. Угол наклона экрана может быть:

```
Горизонтально (0°)          Наклон 45°                Наклон 90°
   ┌──────┐                  ╱──────╲                    │
   │      │  ─── горизонт   │      │  ─── горизонт      │  ─── горизонт
   │телеф.│                 │телеф.│                    │телеф.│
   └──────┘                  ╲──────╱                    │
     земля                    земля                    земля
```

При наклоне телефона оси акселерометра поворачиваются относительно
горизонта. Одна и та же физическая микрораскачка крыла создаёт
разные проекции на оси телефона.

---

## Как сейчас работает

В `SensorController.onLinearAccelSample()`:

```java
// Поворот ускорения из осей телефона в мировые (E=X, N=Y)
worldAccelOut[0] = R[0]*ax + R[1]*ay + R[2]*az;  // восток
worldAccelOut[1] = R[3]*ax + R[4]*ay + R[5]*az;  // север
```

Матрица поворота R строится из:
- Вектора гравитации (TYPE_ACCELEROMETER) — определяет где «верх»
- Вектора магнитного поля (TYPE_MAGNETIC_FIELD) — определяет где «север»

**Если компас есть** — матрица R корректно компенсирует любой наклон.
Оси телефона переводятся в мировые координаты. Детектор видит
горизонтальные колебания независимо от наклона телефона.

**Если компаса нет (fallback)** — поворот не делается, оси телефона
используются как есть. При наклоне сигнал «размазывается» между осями,
амплитуда падает, пороги не срабатывают.

---

## Проблема 1: Падение SNR при наклоне без компаса

Без магнитометра `getRotationMatrix()` возвращает false.
Код падает в:

```java
thermalDetector.processSample(axG, ayG); // оси телефона, не мира
```

При наклоне 45°:
- Горизонтальное колебание амплитудой 0.05g проецируется на оси телефона
  как 0.05g × cos(45°) = 0.035g по каждой оси
- RMS падает в 1.4 раза
- Порог SUSPECT (0.005g) может не сработать

**Решение:** детектор наклона через gravity vector.

---

## Проблема 2: NoiseFloor зависит от наклона

При наклоне шум тела пилота (дыхание, пульс) проецируется на оси иначе.
NoiseFloor, откалиброванный горизонтально, неверен при наклоне.

---

## Алгоритм авто-калибровки наклона

### Шаг 1: Определить угол наклона телефона

При старте, когда пилот висит неподвижно (2 сек), измерить gravity:

```java
// gravityX, gravityY, gravityZ — первые 100 сэмплов TYPE_ACCELEROMETER
float gx = meanGravityX;  // м/с²
float gy = meanGravityY;
float gz = meanGravityZ;
float gMag = sqrt(gx*gx + gy*gy + gz*gz); // должно быть ~9.81

// Угол между осью Z телефона и вертикалью мира
float tiltDeg = (float) Math.toDegrees(Math.acos(gz / gMag));
// 0° = экран вверх, 90° = экран вертикально
```

### Шаг 2: Масштабировать пороги детекции

```java
// Коэффициент: во сколько раз уменьшается проекция горизонтального сигнала
// на оси телефона при данном наклоне
float tiltFactor = (float) Math.cos(Math.toRadians(tiltDeg));
if (tiltFactor < 0.3f) tiltFactor = 0.3f; // защита от 90°

// Скорректированные пороги (ниже = чувствительнее)
float adjustedSuspect = TH_SUSPECT / tiltFactor;
```

### Шаг 3: Детектор получает tiltFactor

Добавить в `processSample()` параметр tiltFactor, либо передавать через
поля SensorController.

## Рекомендуемая процедура калибровки (для пилота)

### При первом запуске — с указанием угла наклона

1. Повесить подвеску на турник
2. Закрепить телефон в штатном положении на груди
3. Запустить TERMO1
4. Нажать **долгим тапом** кнопку СТАРТ → сброс калибровки
5. Висеть неподвижно 3 секунды — автокалибровка измерит наклон
6. На экране появится: `Наклон: 35°`, `Пороги: x1.22`

### Что говорит экран после калибровки

```
Наклон: 35°        ← определён автоматически
Коэфф: x1.22       ← коррекция порогов
Статус: Калибровка OK
```

### Без турника — самопроверка на земле

1. Сидя на стуле, телефон на коленях горизонтально
2. Запустить TERMO1
3. Покачать коленями из стороны в сторону (0.5–1 Гц)
4. Должен появиться жёлтый шарик (режим ТЕРМИК РЯДОМ)
5. Наклонить телефон на 45° — повторить
6. Шарик должен быть примерно той же яркости

## Что изменить в коде

### SensorController

Добавить:

```java
// Определение наклона
private float phoneTiltDeg = 0f;     // угол наклона экрана от горизонта
private float tiltCorrection = 1f;   // множитель порогов (1/cos(tilt))

public float getPhoneTiltDeg() { return phoneTiltDeg; }
public float getTiltCorrection() { return tiltCorrection; }
```

В конце калибровки noiseFloor (первые 100 сэмплов) — вычислить tilt:

```java
if (calibCount == CALIB_SAMPLES) {
    // Вычислить среднюю гравитацию за период калибровки
    float gx = gravitySumX / CALIB_SAMPLES;
    float gy = gravitySumY / CALIB_SAMPLES;
    float gz = gravitySumZ / CALIB_SAMPLES;
    float gMag = (float) Math.sqrt(gx*gx + gy*gy + gz*gz);
    phoneTiltDeg = (float) Math.toDegrees(Math.acos(gz / gMag));
    tiltCorrection = 1f / (float) Math.cos(Math.toRadians(phoneTiltDeg));
    if (tiltCorrection > 3f) tiltCorrection = 3f;
    if (tiltCorrection < 1f) tiltCorrection = 1f;
}
```

### ThermalDetector

Использовать tiltCorrection для адаптации порогов:

```java
// В processSample() — адаптивные пороги с учётом наклона
float tiltFactor = sensorController.getTiltCorrection(); // 1.0..3.0
float suspectThresh  = Math.max(TH_SUSPECT, nf * 3f) / tiltFactor;
float thermalThresh  = Math.max(TH_THERMAL, nf * 5f) / tiltFactor;
float insideThresh   = Math.max(TH_INSIDE,  nf * 10f) / tiltFactor;
```

## Сводка

| Ситуация | Компас есть | Компаса нет |
|----------|-------------|-------------|
| Телефон горизонтально | ✅ Работает | ✅ Работает (оси совпадают) |
| Телефон наклонён 45° | ✅ Матрица R компенсирует | ⚠️ SNR падает в 1.4× |
| Телефон наклонён 90° | ✅ Матрица R компенсирует | ❌ Сигнал почти не виден |

**Вывод:** для работы при наклоне без компаса необходима коррекция
порогов через tiltFactor. Сама коррекция — 3 строки кода.
