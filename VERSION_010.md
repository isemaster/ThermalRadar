# ThermalRadar — Руководство пользователя v0.1.0

Термик-радар для парапланеризма. Анализирует микрораскачку крыла (акселерометр) и барометрический вариометр для обнаружения и центровки термиков.

---

## Новое в версии 0.1.0

### 🔥 Детекция крутки и центровка

**Новый модуль `CirclingManager`** определяет, что пилот крутит термик:

1. **Автоопределение крутки** — по гироскопу (18°/с, подтверждение 4 с)
2. **Экранная надпись «крутим термик»** — появляется золотым по центру после ~540° (1.5 круга), гаснет через 5 с после прекращения вращения
3. **Секторный анализ варио** — 4 сектора по 90° (N/E/S/W). На каждом полном обороте вычисляется, в каком секторе был самый сильный подъём
4. **Голосовые подсказки после каждого круга:**
   - *«ядро на ветер»* — лучший подъём навстречу ветру
   - *«ядро по ветру»* — лучший подъём по ветру
   - *«Ядро север/восток/юг/запад»* — в других направлениях

### 💨 Оценка ветра по дрейфу спиралей

Ветер определяется **по смещению GPS-центров спиралей** (наиболее точный метод для параплана):

- Каждая спираль: сбор GPS-точек → центроид
- После 2+ спиралей: дрейф центра → направление и скорость ветра
- **Стрелка на радаре** — от края круга к центру (показывает, откуда дует ветер)
- **Подпись** — «ветер 3.2 м/с» голубым цветом снаружи радара
- Дополнительно: оценка по GPS-скорости на прямой (воздушная скорость 9.5 м/с)

### 📝 Журнал событий (Events.csv)

В ZIP-чанке появился второй файл — `*_events.csv`. Формат:

```
dtMs,eventType,details
0,WALL_START,<unix_ms>            — привязка к реальному времени
5000,VOICE_SPOKEN,Термик спереди 50 метров
12300,CIRCLING_START,circling confirmed
14000,THERMAL_LABEL_ON,540 deg reached
18500,CIRCLE_GUIDANCE,ядро на ветер
22000,WIND_UPDATE,270deg 3.2m/s
30500,VOICE_SKIP,rear_sector angle=45 rel=120
41200,THERMAL_NEW,bornMs=41200 angle=90 dist=50
45300,THERMAL_REMOVED,bornMs=41200 angle=90 dist=50
```

Все временные метки — по **монотонной шкале** `elapsedRealtime` (без скачков от NTP). Абсолютное время восстанавливается как `wallStartMs + dtMs`.

### 📊 Расширенный набор данных (Samples.csv)

### ⏱ Монотонное время

Все `dtMs` в Samples и Events считаются от `SystemClock.elapsedRealtime()` — монотонная шкала, не зависит от перевода часов или NTP-синхронизации. Временная ось между чанками ровная, без разрывов.

### ☀️ Солнечный режим

В настройках: **«Солнечный режим (яркие цвета)»**:
- Максимальная яркость экрана
- Высококонтрастная цветовая схема HUD
- *(радар пока в стандартных цветах — настройка цветов радара в разработке)*

### 👁 Слепой режим (карман)

Режим «Слепой полёт» в настройках:
- Яркость экрана **15%** (экономия заряда)
- Показывает на тёмном фоне: статус, варио, высоту (MSL + AGL), время полёта
- Основная информация — голосовые подсказки (радар и детектор работают в фоне)

### 🛑 Безопасная остановка записи

- Кнопка **СТОП** — запрос подтверждения «Остановить запись? Да / Нет»
- Автоматической остановки логов по завершению полёта **нет**
- GPS **всегда включён** (координаты пишутся в лог, независимо от настроек)

### 🎯 Исправление порога детекции

- `TH_SUSPECT`: 0.005g → **0.020g** (дистанция обнаружения 150 м → 75 м)
- Добавлен фильтр **SNR > 3** — в снижении (SNR 0.5–1.3) тишина, термик (SNR 5–500) детектируется

## Интерфейс

### Основной экран (радар)

| Элемент | Описание |
|---------|----------|
| **Радар** | Компас, N/S/W/E, круги 50/100/125/150 м. Термики — жёлтые круги, размер/яркость зависят от дистанции |
| **Варио** | По центру вверху: `+2.3 м/с` (зелёный — набор, красный — снижение) |
| **Статус** | Пил: ПОИСК / ПОДОЗРЕНИЕ / ТЕРМИК РЯДОМ / НАБОР / СНИЖЕНИЕ |
| **Высота** | Слева AGL (над землёй), справа MSL (над уровнем моря) |
| **Стрелка ветра** | Голубая, от края к центру радара + подпись скорости |
| **«крутим термик»** | Золотая надпись по центру при крутке >540° |
| **Время** | Время полёта слева, системное время справа внизу |
| **Info** | Панель внизу: уровень турбулентности, SNR, количество блипов |

### Кнопки

| Кнопка | Действие |
|--------|----------|
| **СТАРТ** (слева) | Начать запись лога |
| **СТОП** (справа) | Остановить запись (с подтверждением) |
| **СТАРТ долгий тап** (>0.6 с) | Сброс калибровки |
| **Шестерёнка** (справа вверху) | Настройки |
| **Х** (слева вверху) | Выход |
| **ТЕСТ** (внизу) | Тестовый режим (калибровка осей) |

### Настройки

1. **Sound** — звук варио
2. **Vibration** — вибрация при смене статуса
3. **Слепой полёт (карман)** — экран гаснет до 15%, информация голосом
4. **Голосовые подсказки** — TTS-озвучка направления на термик и центровки
5. **Солнечный режим** — макс. яркость, контрастный HUD
6. **Color scheme** — тёмная / светлая / высокая контрастность
7. **Vario smooth** — сглаживание варио (5–100 отсчётов)
8. **Калибровка** — запоминает угол крепления телефона
9. **Send logs** — выбор и отправка ZIP-логов
10. **Start/Stop Flight Log** — ручной старт/стоп лога (альтернатива кнопкам на экране)
11. **Simulation** — тестовая симуляция полёта

## Структура логов

Логи пишутся в каталог `{ExternalStorage}/logs/`. Нарезка каждые 10 минут:

```
Start20260617120000-20260617121000.zip   (первые 10 мин)
Flight20260617121000-20260617122000.zip  (10–20)
...
Finish20260617150000-20260617151000.zip  (последний)
```

**Внутри каждого ZIP:**

```
Start20260617120000-20260617121000.zip
├── Start20260617120000-20260617121000.csv          — 50 Гц, сенсоры
└── Start20260617120000-20260617121000_events.csv   — события (голос, блипы, крутка)
```

### Формат Samples.csv

```
dtMs,gpsSpeed,gpsHeading,gpsLat,gpsLon,gpsAlt,
gpsFixAge,gpsAccuracy,
vario,
ax,ay,az,gx,gy,gz,
mx,my,mz,pressure,pitch,roll,heading,
thermalAngle,thermalStrength,thermalDist,thermalSource,snr,noiseFloor,
detectStatus
```

- `dtMs` — монотонные мс от старта полёта (elapsedRealtime)
- `gpsFixAge` — возраст последнего GPS fix, мс (растёт при потере сигнала)
- `gpsAccuracy` — точность fix, м (из `Location.getAccuracy()`)
- `vario` — вертикальная скорость, м/с (сглаженная за 750 мс)
- `detectStatus`: 0=SEARCH, 1=SUSPECT, 2=THERMAL, 3=INSIDE

### Формат Events.csv

```
dtMs,eventType,details
0,WALL_START,1750161600000
dtMs,VOICE_SPOKEN,фраза angle=X dist=Y
dtMs,VOICE_SKIP,причина [rear_sector|debounce]
dtMs,THERMAL_NEW,bornMs=X angle=X dist=X strength=X
dtMs,THERMAL_REMOVED,bornMs=X angle=X dist=X strength=X
dtMs,CIRCLING_START,circling confirmed
dtMs,CIRCLING_END,circling stopped
dtMs,THERMAL_LABEL_ON,540 deg reached
dtMs,THERMAL_LABEL_OFF,label hidden
dtMs,WIND_UPDATE,270deg 3.2m/s
dtMs,CIRCLE_GUIDANCE,ядро на ветер
```

### Абсолютное время

Первое событие в первом чанке: `0,WALL_START,<unix_ms>`. Абсолютное время любого события/сэмпла:

```
absTime = wallStartMs + dtMs
```

| Фраза | Условие |
|-------|---------|
| *«Термик слева / спереди слева / спереди / спереди справа / справа X метров»* | Новый подтверждённый термик в передней полусфере |

## Технические детали

- Частота сэмплов: **50 Гц** (LINEAR_ACCELERATION)
- Длина чанка: **10 минут** (ротация по elapsedRealtime)
- Сжатие: **ZIP level 9**
- Объём: ~**400 KB/чанк** (30 000 строк CSV → ZIP)
- За 3 часа полёта: ~**7 MB** (18 чанков)
- Воздушная скорость для оценки ветра: **9.5 м/с** (параплан)
- Порог крутки: **18°/с** по гироскопу, подтверждение **4 с**
- Дебаунс голоса: **8 с**
- GPS: **FusedLocationProvider**, 1 Гц, HIGH_ACCURACY
