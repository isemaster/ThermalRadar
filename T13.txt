## Что осталось исправить

### 1. Аллокации в onDraw (P1 — производительность)

**Проблема:** В `onDraw()` всё ещё создаются объекты в цикле:

```java
// MainActivity.java:2229 — GPS trail conversion
for (double[] pt : gpsTrail) {
    float[] res = new float[2]; // ❌ аллокация для каждой точки!
    Location.distanceBetween(pilotLat, pilotLon, pt[0], pt[1], res);
    // ...
}

// MainActivity.java:2245 — marker conversion
for (double[] pt : entryMarkers) {
    float[] res = new float[2]; // ❌ аллокация для каждого маркера!
    Location.distanceBetween(pilotLat, pilotLon, pt[0], pt[1], res);
    // ...
}

// MainActivity.java:2270 — thermal blips copy
synchronized (thermalLock) {
    thermalsCopy.clear();
    for (ThermalBlip t : thermals) {
        thermalsCopy.add(new ThermalBlip(t)); // ❌ аллокация для каждого blip!
    }
}

// MainActivity.java:2350 — UI text
String labAmp = "±" + String.format(java.util.Locale.US, "%.2f", amp); // ❌ аллокация String + Formatter
String labFreq = String.format(java.util.Locale.US, "%.2fГц", freq); // ❌
String labDir = String.format(java.util.Locale.US, "%.0°", dir); // ❌
```

**Влияние:** ~5000 аллокаций/сек на main thread → GC pauses на старых устройствах.

**Решение:**

```java
// Вынести в поля класса
private final float[] distanceResult = new float[2];
private final StringBuilder textBuffer = new StringBuilder(32);

@Override
protected void onDraw(Canvas canvas) {
    // Переиспользовать буфер
    for (double[] pt : gpsTrail) {
        Location.distanceBetween(pilotLat, pilotLon, pt[0], pt[1], distanceResult);
        float dist = distanceResult[0];
        // ...
    }
    
    // StringBuilder вместо String.format
    textBuffer.setLength(0);
    textBuffer.append("±").append(String.format(java.util.Locale.US, "%.2f", amp));
    String labAmp = textBuffer.toString();
}
```

**Трудозатраты:** 1-2 часа  
**Эффект:** Устранение micro-stutters на 3-4 летних телефонах.

### 2. God Object: MainActivity 2741 строка (P1 — архитектура)

- Sensor wiring

// Предложенная структура

**Трудозатраты:** 2-3 спринта

### 3. POST_NOTIFICATIONS permission (P1 — Android 13+)

**Проблема:** На Android 13+ (API 33+) foreground service notification не покажется без runtime permission `POST_NOTIFICATIONS`.

**Решение:**

```java
// MainActivity.java:250
if (Build.VERSION.SDK_INT >= 33) {
    if (checkSelfPermission(Manifest.permission.POST_NOTIFICATIONS) != GRANTED) {
        requestPermissions(new String[]{Manifest.permission.POST_NOTIFICATIONS}, 1002);
    }
}
```

**Трудозатраты:** 15 минут  
**Эффект:** Корректная работа на Android 13+.

### 4. OSM User-Agent (P2 — безопасность)

**Проблема:** `StaticMapLoader` не устанавливает User-Agent, что может привести к бану IP OSM.

// StaticMapLoader.java
HttpURLConnection conn = (HttpURLConnection) url.openConnection();

**Эффект:** Избежание бана OSM.

### 5. Unit-тесты (P1 — надёжность)

**Проблема:** Критичные алгоритмы (HeadingFilter, CirclingManager, ThermalDetector) не покрыты unit-тестами.

**Требуется минимум:**

```java
// HeadingFilterTest.java
@Test
public void testClamp() {
    HeadingFilter filter = new HeadingFilter();
    filter.update(0, 0);
    double result = filter.update(100, 1); // скачок 100° за 1 мс
    assertTrue(result < 40); // clamp до 40°/с
}

@Test
public void testDeadband() {
    HeadingFilter filter = new HeadingFilter();
    filter.update(0, 0);
    double result = filter.update(0.5, 100); // изменение 0.5° < deadband 1.0°
    assertEquals(0.0, result, 0.01); // не должно измениться
}
```

**Трудозатраты:** 1 спринт  
**Эффект:** Надёжность, предотвращение регрессий.
